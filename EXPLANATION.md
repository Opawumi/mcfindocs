# What is "The Backend" and why do we need it?

Right now, the project is like a **beautifully decorated model home**. When you type a memo and hit "Send," it "acts" like it was sent, but if you refresh the page, it disappears. 

The **Backend** is the "Memory" and "Brain" of the office. We need it for:

1.  **Saving Data (The Database):** So when you write a memo, it stays there forever until deleted.
2.  **Real People (Users):** So users can actually log in with a password and see only their own messages.
3.  **Real File Creation:** Actually turning those memos into real PDF files that people can download and print.
4.  **Security:** Making sure a regular staff member can't go into the Vice-Chancellor's settings and change things.
